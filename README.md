This project focuses on the design and implementation of a hybrid database system for a food delivery platform. It is based on the idea that modern applications require both structured and flexible data handling rather than relying on a single database model. The system supports key operations such as managing customers, restaurants, menus, orders, deliveries, and payments.



To achieve this, a hybrid approach is used:
      -MySQL handles structured, transaction-heavy data like customers, orders, and payments, ensuring accuracy and consistency
      -MongoDB manages flexible and semi-structured data such as menus, allowing the system to adapt easily as data changes
Additional tools support the development process:
     -Draw.io for designing ER diagrams and visualizing the system
     -GitHub for version control and project management
     -Google Docs for organizing and presenting documentation

     
Running the project involves setting up both databases and testing their integration:
     -Install and configure MySQL and MongoDB
     -Create tables (MySQL) and collections (MongoDB)
     -Insert sample data for testing
     -Execute queries to perform operations like placing orders and managing data
     -Validate system performance, accuracy, and integration
     -Generate reports such as order summaries, customer activity, and revenue


Overall, the project demonstrates how combining relational and non-relational databases improves data consistency, flexibility, and scalability, reflecting how real-world systems are designed to handle growing and diverse data needs.
